# Сборка deb пакета из бинарников на linux

## Подготовка окружения

Сперва необходимо убедиться в том, что в среде присутствуют пакеты, необходимые для генерации файлов сборки и утилиты для самой сборки:

```sh
sudo apt update
sudo apt install devscripts build-essential gnupg gpg
```

Если у вас отсутствует сгененрированный довереный GPG ключ для пакетов, то необходимо его создать утилитой gpg:

- Вызвать команду 
    ```sh
    gpg --full-generate-key
    ```

- Ввести все запрашиваемые данные: 
    - Тип ключа: Нажмите 1 (RSA и RSA — по умолчанию).
    - Размер ключа: Введите 4096 (самый безопасный стандарт).
    - Срок действия: Нажмите 0 (ключ не истекает) или выберите нужный срок (например, 1y — один год). Подтвердите выбор, нажав y.
    - Имя (Real name): Введите имя и фамилию (например, Ivan Ivanov). Важно: оно должно точно совпадать с полем Maintainer в вашем файле control.
    - Email: Введите ваш рабочий email. Он также должен совпадать с email в файле control.
    - Примечание (Comment): Можно оставить пустым или написать Astra Linux Developer Key.Подтвердите правильность данных, нажав букву O (или Поехали/Okay).

- Далее получить ключ можно командой
    ```sh
    gpg --list-secret-keys --keyid-format LONG
    ```

    Ключ указан в этой строке:
    <div >sec   rsa4096/<span style="font-weight:bold; font-style: italic">C8DDE55DF7F07056</span> 2026-07-08 [SC]</div> 

## Генерация файлов сборки deb пакета

Для генерации манифеста и остальных файлов используется команда
```sh
 dh_make --native --single -p *имя пакета*
```

Если генерация происходит не из исходников, а для бинарных файлов, то часто необходимо указать какой файл и в какую директорию необходимо поместить при разворачивании проекта.

Для этого создается файл <span style="font-weight:bold; font-style: italic">install</span> и заполняется следующим образом:
```sh
# Директория в которой лежит нужный бинарник | Директория, в который он должен развернуться (с указанием имени конечного исполнительного файла)
bin/snmp_daemon-1.0.0                               usr/bin/snmp_daemon
```

## Генерация пакета

Из папки, где сгенерировлись папка <span style="font-weight:bold; font-style: italic">debian</span> вызвать команду
```sh
debuild -k*ключ*
```

На уровне выше этой директории сформируются все необходимые файлы и сам .deb пакет.
